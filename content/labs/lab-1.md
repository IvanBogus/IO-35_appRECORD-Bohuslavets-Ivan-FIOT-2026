Бізнес-логіка системи полягає в організації зберігання, перегляду та оновлення інформації про студентів і навчальні групи. Система повинна забезпечувати можливість роботи з переліком студентів, прив’язкою студентів до певної групи та поданням інформації у зручному адаптивному інтерфейсі.

Основні правила функціонування системи:
- кожен студент має унікальний ідентифікатор;
- кожен студент повинен належати до певної навчальної групи;
- користувач повинен мати можливість переглядати список студентів;
- користувач повинен мати можливість додавати, редагувати та видаляти записи;

## 2. Реалізація адаптивного інтерфейсу

### 2.1 Структурні компоненти інтерфейсу

Інтерфейс вебзастосунку складається з трьох основних частин:
- `header` — шапка сторінки з логотипом та навігаційним меню;
- `main` — основний блок із контентом та картками студентів;
- `footer` — нижній колонтитул із довідковою та контактною інформацією.

Таке компонування відповідає вимогам семантичної верстки та забезпечує зрозумілу структуру сторінки.

### 2.2 Функціонально-стилістичні рішення

Під час реалізації інтерфейсу було використано такі рішення:
- адаптивна навігація з переходом у бургер-меню на мобільних пристроях;
- адаптивна сітка карток студентів на основі CSS Grid;
- media queries для ширин 1200px, 768px і 480px;
- відносні одиниці вимірювання `%`, `rem`, `vw`, `vh`;
- hover-ефекти для інтерактивних елементів;
- плавне відкриття меню та анімація появи контенту.

## 7. Реалізація серверної частини (Node.js + Express)

### 7.1 Загальний опис

У другій частині лабораторної роботи було реалізовано серверну частину вебзастосунку з використанням середовища Node.js та фреймворку Express.

Сервер забезпечує обробку HTTP-запитів та реалізує базові CRUD-операції для роботи з даними студентів:
- отримання списку студентів (GET);
- додавання нового студента (POST);
- оновлення даних студента (PUT);
- видалення студента (DELETE).

Дані зберігаються у вигляді масиву об'єктів у пам’яті сервера (in-memory storage).

### 7.4 Реалізація CRUD-операцій

#### Початкові дані

```javascript
let students = [
  { id: 1, name: "Іван Петренко", group: "ІО-31" },
  { id: 2, name: "Марія Коваль", group: "ІО-32" }
];
```

#### GET — отримання списку студентів

```javascript
app.get("/students", (req, res) => {
  res.json(students);
});
```

#### POST — додавання студента 

```javascript
app.post("/students", (req, res) => {
  const { id, name, group } = req.body;

  if (!id || !name || !group) {
    return res.status(400).json({
      message: "Потрібно передати id, name, group"
    });
  }

  const exists = students.find(s => s.id === id);

  if (exists) {
    return res.status(409).json({
      message: "Студент з таким id вже існує"
    });
  }

  const newStudent = { id, name, group };
  students.push(newStudent);

  res.status(201).json({
    message: "Студента додано",
    student: newStudent
  });
});
```

#### PUT — оновлення студента

```javascript
app.put("/students/:id", (req, res) => {
  const id = Number(req.params.id);
  const { name, group } = req.body;

  const student = students.find(s => s.id === id);

  if (!student) {
    return res.status(404).json({
      message: "Студента не знайдено"
    });
  }

  if (name) student.name = name;
  if (group) student.group = group;

  res.json({
    message: "Студента оновлено",
    student
  });
});
```

#### DELETE — видалення студента

```javascript
app.delete("/students/:id", (req, res) => {
  const id = Number(req.params.id);

  const index = students.findIndex(s => s.id === id);

  if (index === -1) {
    return res.status(404).json({
      message: "Студента не знайдено"
    });
  }

  const deletedStudent = students[index];
  students.splice(index, 1);

  res.json({
    message: "Студента видалено",
    student: deletedStudent
  });
});
```
